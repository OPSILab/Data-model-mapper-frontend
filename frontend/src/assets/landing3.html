Il Problema dei dati in strutture non standard

Nel panorama aziendale moderno, le aziende producono quotidianamente una quantità enorme di dati. Tuttavia, questi dati sono spesso in formati o strutture ad hoc, impedendo la creazione di valore attraverso la collaborazione inter-aziendale.

Il Data model mapper nasce per superare questa limitazione, consentendo agli utenti di mappare e trasformare i dati in un formato standardizzato e interoperabile.

Smart data model NGSI

Il Data model mapper supporta il modello di dati NGSI, un formato standard aperto che consente la rappresentazione semantica dei dati. Questo modello è progettato per essere flessibile e scalabile, facilitando l'integrazione e la condivisione dei dati tra diverse piattaforme e applicazioni.

Come Funziona il Data model mapper

Il Data model mapper consente agli utenti di definire le regole di mappatura tra i loro dati e il modello (eventualmente) NGSI. Attraverso un'interfaccia intuitiva, gli utenti possono selezionare le entità e gli attributi da mappare, definire le trasformazioni necessarie e generare automaticamente il modello NGSI. I formati di input supportati includono JSON, GEOJSON, CSV, rendendo il processo di integrazione dei dati semplice e veloce. I formati di output possono essere JSON e GEOJSON. Gli input possono provenire dal suo DB interno, da un file locale, da un URL esterno o da Minio. L'output può essere scaricato o inviato su Orion. Il data model mapper supporta 3 modalità di funzionamento:

  • Modalità web app: gli utenti possono caricare i loro dati tramite una UI e mappare le entità e gli attributi in modo interattivo.
  • Modalità CLI: gli utenti possono utilizzare NodeJS per invocare uno script javascript di mappatura tramite prompt dei comandi
  • Modalità API: il Data model mapper offre un'API RESTful per il processo di mappatura dati, a cui si appoggia la web app

Tutte e 3 le modalità condividono il medesimo funzionamento

  1. Impostazione configurazione: Definisci le opzioni di mappatura e trasformazione.
  2. Selezione formato e sorgente di input: Scegli tra file locale, URL, DB interno o Minio.
  3. Caricamento file sorgente: Importa i dati da trasformare.
  4. Scelta o caricamento schema di output: Seleziona o carica lo schema che rappresenta la struttura finale.
  5. Definizione file di mappatura: Collega input e output tramite regole personalizzate.
  6. Avvio trasformazione: Esegui la conversione dei dati.
Risultato: scaricabile in JSON o GEOJSON, oppure inviabile direttamente su Orion.
Salvataggio: tutti i dati e i passaggi possono essere salvati sul DB interno o esportati in JSON per riprendere il lavoro.
Condivisione: il file di mappatura, caricato su Minio, è condivisibile con altri utenti.
Automazione: la modalità UI genera automaticamente una richiesta curl e il body per l’API, facilitando integrazione e automazione.
